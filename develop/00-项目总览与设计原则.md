## 当前实现边界

当前仓库主链路为纯 C++ 架构：
- `core_engine`：行情接入、状态计算、风控、执行、回报落库。
- `StrategyEngine`：进程内策略调度与回调分发。
- 存储层：`IRealtimeCache` / `ITimeseriesStore` 抽象 + in-memory/external 双模式。
- 运维与验收：C++ CLI + Shell 编排，证据输出到 `docs/results/`。

当前交付口径优先单机生产 SLA 基线（发布、回滚、恢复、证据闭环），多机主备在同一协议下演进。

## 设计原则（当前有效）

1. 热路径以确定性、可回放、可测性优先。
2. 策略运行统一使用 `ILiveStrategy` 接口与 `StrategyRegistry` 注册机制。
3. 文档必须可由仓库路径和命令直接验证。
4. 任何新增能力进入主链路前，必须补齐测试与回归门禁。

## 迭代目标（仓库可实施）

1. 研究与回测 CLI 完整化（可复现实验、统一报告、一致性门禁）。
2. 风控与执行策略增强（规则化、可解释、可审计）。
3. 运维闭环强化（演练频率、证据完整性、阈值门禁）。

## 落地依赖条件

- 合约字段与配置键保持稳定，避免跨模块漂移。
- 外部依赖（Redis/Timescale/CTP）需提供稳定可达与凭据管理方案。
- 演练与验收模板统一，保障跨环境可比性。
